Подготовительная информация:

Перед началом создания файла необходимо уточнить несколько параметров:

- Получить IP сервера на котором будет установлен Sonar
- Определить какая временная база 1С будет использоваться. Для работы gitsync необходимо создание тестовой базы 1С. gitsync умеет работать с файловой базой (которую создает сам) и с серверной базой. Файловую базу необходимо использовать, если на компьютере, на котором будет установлен Sonar server есть доступ к ключу 1С, то есть на этом компьютере Вы можете создать файловую базу и зайти в неё. Если на компьютере с Sonar нет доступа к лицензиям, то необходимо использовать серверную базу (которую необходимо предварительно создать)
- Получить полный путь к хранилищу

Установка Docker:

Для формирования докера необходимо сформировать Архивный файл (выбрав необходимые параметры ниже).
Далее перенести архив на сервер (на котором обязательно должен быть установлен Docker).
Распаковать архив.

В хранилище создать пользователя deploy с паролем deploy

Далее:
Запустить .init_gitsync - папке 1C_GIT будет создан файл VERSION,  в случае, если если вы начинаете использовать Sonar не с первого дня разработки рекомендуем изменить версию на предпоследнюю версию в истории хранилища.
Например указать 4


Если файл не создан, то следующие шаги выполнять не рекомендуется.

Запустить .init_sonar_server  Если Sonar был запущен, то вы должны получить доступ в консоль по адресу http://IPссервера:9000/ 

Административный доступ: admin пароль adminsonar
Пользовательский доступ: tedo пароль tedo

Для запуска gitsync запустите sync_gitsync -  после этого в папке 1C_Git должна быть создана выгрузка объекто
Для  запуска сканера Sonar запустите -  start_sonar_scaner - после этого в проекте TeDo на http://IPссервера:9000/  будет доступна информация об ошибках

Для автоматического запуска синхронизации настройте Jenkins (следующие шаги)
